Create two more route handlers. The wallet route is a dynamic segment (more complex — has aggregation logic). The LP fees route combines two lib/api functions.

**src/app/api/wallet/[address]/route.ts:**
- `export const revalidate = 300` (tx history — 5 min literal)
- Import `NextResponse` from `next/server`, `NextRequest` type from `next/server`
- Import `getAllAccountTransfers` from `@/lib/api/solscan`
- Import `WalletActivityResponse` type from `@/lib/api/types`
- CRITICAL: `params` is a Promise in Next.js 16 — must `await params`
- Function signature: `export async function GET(_request: NextRequest, { params }: { params: Promise<{ address: string }> })`
- Inside: `const { address } = await params`
- Fetch in and out transfers in parallel: `Promise.all([getAllAccountTransfers(address, { flow: 'in' }), getAllAccountTransfers(address, { flow: 'out' })])`
- Convert lamports to SOL: `const LAMPORTS = 1_000_000_000`
- Build transactions array: map both inTransfers and outTransfers to `{ txHash: t.trans_id, timestamp: t.block_time, amount: t.amount / LAMPORTS, flow: 'in' | 'out' as const }`, spread both, sort by timestamp descending
- Compute `totalIn` and `totalOut` by reducing each array (sum of `t.amount / LAMPORTS`)
- Construct `WalletActivityResponse`: `{ totalIn, totalOut, netBalance: totalIn - totalOut, transactions }`
- Return `NextResponse.json(data)`
- Error: `console.error('[wallet API] ${address}', err)` then 500
- This route serves R6 (marketing wallet), R7 (dev wallet), and R8 (LP wallet) — same endpoint, different address param

**src/app/api/lp-fees/route.ts:**
- `export const revalidate = 300` (fee data — 5 min literal)
- Import `getLpFeeInflows` and `getFeeDistribution` from `@/lib/api/reflections`
- Import `LpFeeResponse` type from `@/lib/api/types`
- GET handler:
  - `const { totalFeeSol, inflows } = await getLpFeeInflows()`
  - `const distribution = getFeeDistribution(totalFeeSol)` (pure function, no await needed)
  - Construct `LpFeeResponse`: `{ totalFeeSol, inflows, distribution }`
  - Return `NextResponse.json(data)`
- Error: `console.error('[lp-fees API]', err)` then 500
- Note: `inflows` from `getLpFeeInflows()` returns objects with `{ txHash, timestamp, amountSol, fromAddress }`. The `LpFeeResponse` type expects `{ txHash, timestamp, amountSol }` — map the inflows to drop `fromAddress` before returning, OR verify the extra field does not cause a type error (TypeScript allows extra properties in object literals passed directly, but it may flag it in explicit typing). If needed, map: `inflows.map(({ txHash, timestamp, amountSol }) => ({ txHash, timestamp, amountSol }))`
