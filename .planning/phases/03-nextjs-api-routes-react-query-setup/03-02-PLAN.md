Create three React Query v5 hooks. Each file must start with `'use client'` directive (these are client components). All hooks follow the same pattern: export a named function that returns `useQuery(...)` with a single options object (v5 API — NOT the deprecated 3-arg form).

Every hook imports its response type from `@/lib/api/types` (NOT from lib/api/ composite modules — those are server-side only). Every hook uses `fetch('/api/...')` to call the API route — NEVER import lib/api/ functions directly (they use process.env server-side keys).

**src/hooks/usePrice.ts:**
```
'use client'
import { useQuery } from '@tanstack/react-query'
import type { PriceResponse } from '@/lib/api/types'
```
- Export `usePrice()` function
- `queryKey: ['price']`
- `queryFn`: fetch `/api/price`, check `res.ok`, throw `new Error('Price fetch failed')` if not, return `res.json() as Promise<PriceResponse>`
- `staleTime: 30_000` (30s — prices feel live, lower than global 60s default)
- `refetchInterval: 60_000` (60s background refresh per R11 requirement)
- Type the useQuery generic as `useQuery<PriceResponse, Error>(...)`

**src/hooks/useTreasury.ts:**
